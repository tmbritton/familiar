# Story 6.5-2: Tighten Typespecs and Remove Dead Clauses

Status: ready-for-dev

## Story

As a developer maintaining the Familiar codebase,
I want all `@spec` annotations to accurately reflect function return types and all unreachable pattern match clauses removed,
so that Dialyzer passes clean and the pre-commit hook can enforce it.

## Context

After fixing `unknown_type` warnings (Story 6.5-1), ~30 Dialyzer warnings remain:
- 16 `contract_supertype` — spec is broader than actual return type
- 7 `extra_range` — spec includes types the function never returns
- 4 `pattern_match_cov` — dead clauses covered by prior patterns
- 1 `pattern_match` — pattern can't match the actual type
- 1 `no_return` — `System.halt` in escript entry (expected)
- 1 `call` — function call type mismatch

Once all warnings are resolved, Dialyzer should be promoted from advisory to blocking in the pre-commit hook.

### Approach

For `contract_supertype`: Dialyzer reports the actual "success typing" — use that directly as the new spec. Don't over-narrow; keep the spec accurate for future callers.

For `extra_range`: Functions like `Work.update_status/2` spec `{:ok, map()} | {:error, ...}` but only ever return `{:error, ...}` (not_implemented stub). Remove the `{:ok, ...}` until implemented.

For dead clauses: These are clauses where a prior clause already matches all possible values of the type. Safe to delete.
