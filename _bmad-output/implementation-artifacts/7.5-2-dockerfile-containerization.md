# Story 7.5-2: Dockerfile for Containerized Execution

Status: done

## Story

As a developer running Familiar,
I want to run agents inside a Docker container with only the project codebase mounted as a shared volume,
so that agent file operations and shell commands are sandboxed at the OS level.

## Context

Agents have real filesystem access via `write_file`, `delete_file`, and `run_command` tools. The safety extension sandboxes writes to the project directory, but a container adds OS-level enforcement. The Familiar source code stays inside the container — only the target project is shared.

## Acceptance Criteria

### AC1: Dockerfile Builds a Working Familiar Image

**Given** the Familiar source code
**When** `docker build -t familiar .` runs
**Then** the image contains the compiled Elixir app, escript, and all dependencies
**And** the image does NOT contain the target project codebase

### AC2: Container Mounts Project as Shared Volume

**Given** a target project directory on the host
**When** the container runs with `-v /path/to/project:/project`
**Then** Familiar can read/write files in `/project`
**And** `.familiar/` config, roles, skills, workflows, and SQLite DB persist on the host via `/project/.familiar`
**And** no other host directories are accessible from the container

### AC3: `fam` Commands Work via docker exec

**Given** a long-running daemon container named `fam`
**When** the user runs `docker exec -it fam fam chat`
**Then** `fam chat` connects to the running OTP app
**And** all CLI commands work normally
**And** a host alias `fam='docker exec -it fam fam'` makes it feel native

### AC4: Environment Variables Pass Through

**Given** API keys for remote LLM providers
**When** the container runs with `-e FAMILIAR_API_KEY=xxx`
**Then** the provider adapter reads the key from the container's environment

### AC5: Docker Compose for Convenience

**Given** a `docker-compose.yml`
**When** the user runs `docker compose run fam chat`
**Then** volumes and env vars are pre-configured
**And** the project directory is mounted automatically

## Dev Notes

### Volume Architecture

```
Host machine:
  /home/user/my-project/          ← user's codebase
    .familiar/                     ← persisted config + DB
      config.toml
      familiar.db
      roles/
      skills/
      workflows/
    src/
    lib/
    ...

Container:
  /app/                            ← Familiar source (built into image, NOT shared)
    familiar/
      lib/
      _build/
      ...
  /project/                        ← mounted from host (-v)
    .familiar/                     ← read-write, persists on host
    src/
    lib/
    ...
```

### Dockerfile Sketch

```dockerfile
FROM hexpm/elixir:1.15.7-erlang-26.2.1-debian-bookworm-20240130-slim AS build

RUN apt-get update && apt-get install -y git sqlite3 libsqlite3-dev build-essential

WORKDIR /app
COPY familiar/ ./

RUN mix local.hex --force && mix local.rebar --force
RUN mix deps.get --only prod
RUN MIX_ENV=prod mix compile
RUN MIX_ENV=prod mix escript.build

FROM debian:bookworm-slim AS runtime

RUN apt-get update && apt-get install -y libsqlite3-0 git && rm -rf /var/lib/apt/lists/*

COPY --from=build /app/fam /usr/local/bin/fam

ENV FAMILIAR_PROJECT_DIR=/project
WORKDIR /project

ENTRYPOINT ["fam"]
```

### docker-compose.yml Sketch

```yaml
services:
  fam:
    build: .
    volumes:
      - .:/project
    environment:
      - FAMILIAR_API_KEY=${FAMILIAR_API_KEY}
      - FAMILIAR_BASE_URL=${FAMILIAR_BASE_URL:-https://api.deepseek.com}
      - FAMILIAR_CHAT_MODEL=${FAMILIAR_CHAT_MODEL:-deepseek-chat}
    stdin_open: true
    tty: true
    restart: unless-stopped
```

### Architecture: Long-Running Container + docker exec

The container runs as a persistent daemon. CLI commands reach it via `docker exec`:

```
Host machine                          Container (long-running)
┌──────────────┐                     ┌──────────────────────┐
│ alias fam=   │  docker exec -it    │  /app/               │
│ 'docker exec │ ──────────────────► │    Familiar OTP app  │
│  -it fam fam'│                     │    Phoenix daemon     │
│              │                     │    AgentSupervisor    │
│ /my-project/ │ ◄── shared volume ──│  /project/           │
│   .familiar/ │                     │    .familiar/         │
│   src/       │                     │    src/               │
└──────────────┘                     └──────────────────────┘
```

### Usage

```bash
# === First-time setup ===

# Build the image
docker build -t familiar .

# Start the daemon container (runs in background)
docker run -d --name fam \
  -v $(pwd):/project \
  -e FAMILIAR_API_KEY=sk-xxx \
  -e FAMILIAR_BASE_URL=https://api.deepseek.com \
  -e FAMILIAR_CHAT_MODEL=deepseek-chat \
  familiar daemon start

# Add alias to shell profile (~/.bashrc or ~/.zshrc)
echo 'alias fam="docker exec -it fam fam"' >> ~/.bashrc
source ~/.bashrc

# === Daily use — feels native ===

fam init                           # initialize project
fam chat                           # interactive conversation
fam plan "Add user auth"           # run planning workflow
fam roles                          # list roles
fam validate                       # validate config
fam sessions                       # list sessions
fam chat --role analyst            # chat with specific role
fam chat --json                    # programmatic mode

# === Lifecycle ===

docker stop fam                    # stop the daemon
docker start fam                   # restart it
docker logs fam                    # view daemon logs
docker rm -f fam                   # remove container

# === Switch projects ===
# Stop, re-run with different volume mount
docker rm -f fam
cd /other/project
docker run -d --name fam -v $(pwd):/project -e FAMILIAR_API_KEY=xxx familiar daemon start
```

### Paths.project_dir Change

Currently:
```elixir
def project_dir do
  Application.get_env(:familiar, :project_dir, File.cwd!())
end
```

Needs to also check `FAMILIAR_PROJECT_DIR` env var:
```elixir
def project_dir do
  Application.get_env(:familiar, :project_dir) ||
    System.get_env("FAMILIAR_PROJECT_DIR") ||
    File.cwd!()
end
```
