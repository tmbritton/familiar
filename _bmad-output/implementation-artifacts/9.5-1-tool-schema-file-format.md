## Dev Notes

### Current @schemas map (tool_schemas.ex lines 9‚Äì158)

13 tools, each with `description` (string) and `parameters` (OpenAI JSON Schema object). Key shapes:

- **Simple required param:** `read_file` ‚Äî one required `"path"` string
- **Multiple required params:** `write_file` ‚Äî `"path"` and `"content"` both required
- **Optional params:** `list_files` ‚Äî `"path"` is optional (no `"required"` key)
- **No params:** `monitor_agents`, `signal_ready` ‚Äî empty `"properties"` map
- **Multi-line description:** `store_context` type field has a long description string

### TOML format design

```toml
name = "read_file"
description = "Read the contents of a file at the given path"

[parameters]
type = "object"
required = ["path"]

[parameters.properties.path]
type = "string"
description = "Absolute or relative file path to read"
```

TOML naturally maps nested tables to nested maps. `Toml.decode/1` returns string keys. The `parse_toml/1` function needs to:
1. Extract `name` and `description` from top level
2. Extract `parameters` table ‚Äî already has `type`, `properties`, optional `required`
3. Return `%{description: description, parameters: parameters_map}` with string keys in parameters (matching current `@schemas` shape)

### DefaultFiles file extension filter

Currently at line 19: `String.ends_with?(filename, ".md")`. Change to accept both `.md` and `.toml`:
```elixir
String.ends_with?(filename, ".md") or String.ends_with?(filename, ".toml")
```

Or use: `Path.extname(filename) in ~w(.md .toml)`

### Toml library usage in project

`Toml.decode_file/1` is used in `Familiar.Config` (line 70). The library returns string-keyed maps. Example:
```elixir
Toml.decode(toml_string)
# => {:ok, %{"name" => "read_file", "description" => "...", "parameters" => %{...}}}
```
