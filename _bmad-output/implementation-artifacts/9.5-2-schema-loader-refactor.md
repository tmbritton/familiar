# Story 9.5-2: Schema Loader and ToolSchemas Refactor

Status: done

## Story

As a Familiar user,
I want `ToolSchemas` to load parameter definitions from `.familiar/tools/` at startup,
So that I can customize tool descriptions and parameters for my domain.

## Acceptance Criteria

1. **AC1: Runtime schema loading.** `ToolSchemas.load/1` scans `.familiar/tools/*.toml`, parses each via `parse_toml/1`, and stores the results for runtime queries.

2. **AC2: Fallback to compiled defaults.** For any tool that has a compiled-in default (in `priv/defaults/tools/`) but no custom file in `.familiar/tools/`, the default is used automatically.

3. **AC3: `:persistent_term` storage.** Loaded schemas are stored via `:persistent_term` for zero-cost reads. `for_tools/1` and `all/0` read from `:persistent_term` instead of the compile-time `@schemas` attribute.

4. **AC4: `@schemas` removed.** The hardcoded `@schemas` map is deleted from `ToolSchemas`. TOML files (loaded or compiled-in defaults) are the sole source of truth.

5. **AC5: Malformed file handling.** If a `.toml` file in `.familiar/tools/` fails to parse, a warning is logged and the compiled-in default for that tool is used instead. If no default exists, the tool simply has no schema.

6. **AC6: Startup integration.** `ToolSchemas.load/1` is called during application startup (in `load_extensions/0` or similar), after DefaultFiles has installed defaults but before agents can run.

7. **AC7: Tests — custom override.** Test that a custom `.familiar/tools/read_file.toml` with a modified description overrides the default.

8. **AC8: Tests — fallback.** Test that when no custom file exists, the compiled-in default is used.

9. **AC9: Tests — malformed file.** Test that a malformed TOML file logs a warning and falls back to the default.

10. **AC10: Tests — existing callers.** All existing `for_tools/1` and `all/0` tests pass without modification (behavior is identical after loading defaults).

11. **AC11: Clean toolchain.** `mix compile --warnings-as-errors`, `mix format --check-formatted`, `mix credo --strict`, `mix test`, `mix dialyzer` all pass.

12. **AC12: Stress-tested.** New and modified tests pass 50 consecutive runs with no flakes.

## Dev Notes

### Current ToolSchemas architecture

Pure module with compile-time `@schemas` map (atom-keyed, 13 entries). `for_tools/1` takes string tool names, converts to atoms via `safe_to_atom/1`, looks up in `@schemas`. `all/0` iterates `@schemas` keys.

### Target architecture

`:persistent_term`-backed module. `load/1` populates `:persistent_term` with a string-keyed map. `for_tools/1` does direct `Map.get/2` on the string-keyed map (no atom conversion needed). `all/0` iterates the map.

Why `:persistent_term` over GenServer:
- Zero-cost reads — no message passing, no serialization
- `for_tools/1` is called on every agent prompt assembly (hot path)
- Schemas change only at startup (no concurrent writes)
- Already used in the codebase (`Application` drift warnings)
- No supervision needed — survives process crashes

### `:persistent_term` key

```elixir
:persistent_term.put({Familiar.Execution.ToolSchemas, :schemas}, schemas_map)
:persistent_term.get({Familiar.Execution.ToolSchemas, :schemas}, %{})
```

The second arg to `get/2` defaults to `%{}` so `for_tools/1` works even if `load/1` hasn't been called yet (returns empty schemas, not a crash).

### `load/1` algorithm

```elixir
def load(familiar_dir) do
  tools_dir = Path.join(familiar_dir, "tools")

  # 1. Load all compiled-in defaults as baseline
  defaults = load_compiled_defaults()

  # 2. Overlay custom files from .familiar/tools/
  customs = load_custom_files(tools_dir)

  # 3. Merge: custom overrides default
  merged = Map.merge(defaults, customs)

  # 4. Store
  :persistent_term.put({__MODULE__, :schemas}, merged)
  :ok
end
```

### Loading compiled-in defaults

Iterate all `.toml` files in `DefaultFiles` "tools" subdir. Use `DefaultFiles.default_content("tools", filename)` to get compiled-in content, then `parse_toml/1`.

Note: `DefaultFiles` doesn't expose a list of filenames per subdir. Options:
1. Add `DefaultFiles.list_files(subdir)` — returns filenames for a subdir
2. Scan `priv/defaults/tools/` at compile time in ToolSchemas
3. Use the `@defaults` data by adding a query function to DefaultFiles

Option 1 is cleanest. Add:
```elixir
def list_files(subdir) do
  @defaults
  |> Enum.filter(fn {s, _, _} -> s == subdir end)
  |> Enum.map(fn {_, f, _} -> f end)
end
```

### Test setup for `:persistent_term`

Tests that call `for_tools/1` or `all/0` need schemas loaded. Options:
1. Call `ToolSchemas.load/1` in each test's setup with a temp dir
2. Add a `ToolSchemas.load_defaults/0` that loads only compiled-in defaults (no disk)

Option 2 is simpler for tests that don't need custom overrides. The existing `for_tools/1` and `all/0` tests just need defaults loaded.

### Refactored `for_tools/1`

```elixir
def for_tools(tool_names) do
  schemas = :persistent_term.get({__MODULE__, :schemas}, %{})

  tool_names
  |> Enum.map(fn name ->
    case Map.get(schemas, name) do
      nil -> nil
      schema -> build_openai_schema(name, schema)
    end
  end)
  |> Enum.reject(&is_nil/1)
end
```

No more `safe_to_atom/1` — string keys throughout.
