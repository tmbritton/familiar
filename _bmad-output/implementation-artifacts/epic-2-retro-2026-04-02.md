- **Embed-before-persist pattern.** Code review caught ordering bug in Story 2.2 ŌĆö persisting before embedding left entries without vectors on crash. Became the standard for all subsequent stories (2.3, 2.4, 2.6).
- **Defense-in-depth secret filtering.** SecretFilter runs at extraction, at the storage gateway (`store/1`, `update_entry/2`), and is idempotent. Multiple layers ensure secrets are caught even if one is bypassed.
- **Fail-open philosophy.** Freshness, hygiene, and recovery all log warnings and continue rather than blocking. Correct default for developer tools.

## Challenges

- **Unsupervised background tasks.** `Task.start` in `trigger_background_maintenance` caused recurring test interference across stories 2.2, 2.3, 2.7. Background refresh consumed mock expectations, escaped Ecto sandbox, and forced global stub patterns. Root cause: no `Task.Supervisor` in supervision tree.
- **Time-dependent test fragility.** Freshness tests in 2.7 initially passed or failed depending on wall clock time. `entry.updated_at` used real system clock while mtime was hardcoded. Resolved by DI clock, but burned tokens on the far-future date hack first.
- **N+1 query patterns accumulated.** Deferred in 2.4 (`find_consolidation_candidates`), 2.4 (`load_existing_entries`), and 2.5 (`compute_staleness_ratio`). Each individually acceptable for MVP, but three N+1 patterns is a trend.
- **Backup/restore in-process limitation.** SQLite connection pool retains in-memory state after file-level restore. Can't verify data consistency after restore via `Repo.query` ŌĆö tests only prove file operations work.

## Key Insights

1. **Time-dependent tests = DI the clock.** When tests depend on wall clock time, inject the clock behaviour. Don't pick clever dates or add sleep. The Clock behaviour port existed since 2.2 but wasn't wired into Entry timestamps until Buddy flagged it in 2.7.
2. **Background tasks need supervision.** Both for production resilience (crashed tasks silently lost) and test isolation (tasks escape Ecto sandbox). `Task.Supervisor` is the fix.
3. **Code review catches ordering bugs humans miss.** Embed-before-persist was discovered in review and became the standard pattern. 32 patches across 7 stories ŌĆö the review cycle is high-value.
4. **Fail-open is the right default for developer tools.** Log warnings, return success, don't block the user. Applied consistently across freshness, hygiene, and recovery.
5. **Deferred items accumulate.** Three N+1 patterns, two TOCTOU races, two secret filter gaps ŌĆö individually minor, collectively a trend. Address between epics before they compound.
6. **Pay-it-forward debt cleanup works.** Epic 1's 9-item cleanup prevented problems in Epic 2. Committing to 24 items before Epic 3.

## Significant Discovery: Librarian Agent Pattern

During the retrospective, Buddy identified an architectural enhancement for knowledge retrieval: a **librarian agent** that mediates all knowledge store queries.

**Concept:** A GenServer that receives queries, runs `Knowledge.search/1`, evaluates results, autonomously re-queries when initial results suggest further relevant entries, and returns curated, summarized context.

**Key Design Decisions:**
- Woven into Epic 3 Story 3-1 (Planning Conversation Engine), not a separate story
- Serves both agent queries and user queries (`fam search`)
- `fam search` defaults to librarian-curated results; `--raw` flag for unfiltered results
- Multi-hop retrieval: evaluate ŌåÆ detect gaps ŌåÆ re-query with refined terms ŌåÆ synthesize (all within single query lifecycle)
- **Ephemeral, not stateful:** spins up per query, shuts down after delivering results. Fresh context each time ŌĆö no stale state, bounded memory

**Architecture Fit:**
- Ephemeral GenServer under DynamicSupervisor (consistent with agents-as-actors pattern ŌĆö core reason BEAM/OTP was chosen)
- Spins up on query, performs search ŌåÆ evaluate ŌåÆ optional re-query ŌåÆ summarize ŌåÆ deliver ŌåÆ terminate
- Wraps `Knowledge.search/1` with LLM summarization via existing LLM behaviour port
- Uses existing infrastructure: Embedder (search), Knowledge (retrieval), LLM (summarization)

**Motivation:** Limit agent working context to save tokens. Raw search returns all ranked entries; librarian curates to only what's relevant, with citations. Token savings compound across all epics that query knowledge.

**Impact on Epic 3:**
- Story 3-1 scope expands to include librarian agent as the knowledge query interface
- Story 3-4 (Spec Review & Approval CLI) ŌĆö `fam search` gets `--raw` flag
- Epics file should be reviewed to incorporate this pattern before story creation

**Epic Update Required:** YES ŌĆö Story 3-1 scope review before creating story file.
