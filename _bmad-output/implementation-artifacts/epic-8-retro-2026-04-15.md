## Successes

1. **Pure-module-first architecture.** Protocol and Dispatcher (Story 8-1) are pure functions with zero process state. This made every downstream story easier — Client, Extension, CLI, and integration test all built cleanly on top.

2. **`port_opener` DI pattern unlocked testability.** One function injection in Story 8-2 swapped real OS ports for FakePort GenServers. Reused across 3 test files and the integration test. Zero subprocess management complexity in tests.

3. **Code review caught 22 real bugs.** Not style nits — production-impacting issues: OptionParser flags silently dropped (8-4), bare match crash on failure (8-4), missing COMMAND column (8-4), JSON field type-checking (8-3), connect_timer race (8-2).

4. **Layered story progression with zero rework.** 8-1 codec → 8-2 client → 8-3 storage → 8-4 CLI → 8-5 integration. No story had to go back and fix something from an earlier one.

5. **Zero flaky tests.** 250 consecutive stress-test runs (50 per story) with zero failures. Testing discipline from Epics 5.5 and 7 is now self-sustaining. The `@moduletag :tmp_dir` pattern (vs `System.tmp_dir!()`) caught proactively in 8-4 before it became a flake.

## Challenges

1. **`best_effort_reload` design gap in integration tests.** CLI commands fire `MCPClient.reload_server/1` without test infrastructure opts, spawning real subprocesses that immediately die. The integration test needed branching cleanup logic (`if Process.alive?` → terminate vs manually unregister tools). Works but is the most complex part of the test.

2. **Credo cyclomatic complexity limit forced excessive extraction in 8-4.** Six helpers extracted purely to stay under the limit of 9. Some are cleaner code; some are just satisfying the linter.

3. **`changeset_to_mcp_error` substring matching.** Dispatches on human-readable validation messages ("already been taken", "fam_"). Fragile but no better option until Ecto provides machine-readable error tags.

4. **ETS table ownership.** Extension loader process owns `:familiar_mcp_servers`, not a long-lived process. Single-threaded init makes this safe today, but it's a latent risk.

5. **`main.ex` continues to grow.** Now ~2200+ lines with 7 new MCP subcommands. Testable via DI but hard to navigate.

## Key Insights

1. **Layered pure-to-stateful architecture works.** Protocol (pure) → Client (GenServer) → Extension (lifecycle) → CLI (dispatch). Each layer testable independently. Integration test needed only one fake.

2. **DI injection points compound.** `port_opener`, `tool_registry`, `supervisor`, `deps` map — each added for one story's tests, reused by every subsequent story.

3. **Code review catches real bugs, not style nits.** 22 patches = 22 prevented production bugs. The three-agent adversarial review (Blind Hunter, Edge Case Hunter, Acceptance Auditor) continues to justify its cost.

4. **FakePort is a reusable test primitive that needs a home.** Three identical definitions across test files. Stable enough to extract.

5. **Deferred items are healthy scoping.** 25 items sounds high but most are "not this story's scope" rather than "too hard to fix." `deferred-work.md` tracks them with context.

6. **Testing discipline is self-sustaining.** The patterns (async:false, tmp_dir, stress test) are now automatic. No flaky test debugging this entire epic.

## Patterns Established

1. **Pure codec modules** — encode/decode as plain functions, no process state
2. **`port_opener` DI** — `fn cmd, args, env -> {port_ref, send_fn, close_fn} end` injection for testing
3. **FakePort GenServer** — test double for MCP server subprocess (needs extraction to test/support/)
4. **ETS extension state** — `{name, source, pid}` tuples for queryable, mutable extension state
5. **`best_effort_reload`** — mutations always succeed even if daemon/client unavailable
6. **CLI `parse_args` → `run`** — test the real OptionParser path for flag coverage
7. **Changeset → error atom mapping** — `changeset_to_mcp_error/1` for CLI-friendly error atoms
8. **Config warn-and-skip** — invalid config.toml entries log warnings, don't crash boot
