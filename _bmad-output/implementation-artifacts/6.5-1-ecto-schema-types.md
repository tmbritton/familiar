# Story 6.5-1: Add @type t to Ecto Schemas

Status: done

## Story

As a developer maintaining the Familiar codebase,
I want all Ecto schemas to export `@type t` definitions,
so that Dialyzer can verify typespec contracts across module boundaries.

## Context

Dialyzer reports 23 `unknown_type` warnings because `Conversation.t()`, `Message.t()`, and `Entry.t()` are referenced in `@spec` annotations but never defined. Ecto schemas don't auto-generate a `t` type — it must be declared explicitly.

## Acceptance Criteria

### AC1: All Ecto Schemas Define `@type t`

**Given** each Ecto schema module (Conversation, Message, Entry)
**When** `@type t :: %__MODULE__{}` is added
**Then** Dialyzer resolves all 23 `unknown_type` warnings referencing these schemas

### AC2: Test Baseline

**Given** all changes are applied
**When** `mix test` and `mix compile --warnings-as-errors` run
**Then** 0 failures, 0 compile warnings

- [x] Task 2: Add `@type t :: %__MODULE__{}` to Message schema
