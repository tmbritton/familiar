## Successes

1. **CLI-first architecture proved its value.** Every capability is accessible via `fam <command> --json`, making Familiar usable both as a standalone tool and as a programmatic backend for Claude Code or other agents. The `fam chat` command with `--json` mode enables agent-to-agent integration without HTTP.

2. **The DI pattern scaled cleanly across 20+ commands.** Every CLI command uses the same `deps` map pattern for testability. New commands are mechanical: add a `run_with_daemon` clause, a text formatter, a quiet_summary, and tests. The pattern is so consistent that code reviews for Stories 7-2 through 7-5 found zero issues.

3. **Dialyzer investment paid for itself immediately.** Adding Dialyzer caught 3 dead clauses in the Story 6.5-2 commit and 1 missing `:unexpected_value` variant in the hooks error reporter during the flaky test fix. Both would have been runtime bugs.

4. **Pre-commit hook prevents regressions.** 6 checks (compile warnings-as-errors, format, credo, sobelow, tests, dialyzer) run on every commit. Caught 4 issues during the sprint that would have been committed otherwise: alias ordering, nesting depth, format drift, and the Dialyzer `:unexpected_value` clause.

5. **Flaky test elimination was thorough.** Root causes identified and fixed at the source:
   - Hooks: replaced `capture_log` with `on_handler_error` DI callback
   - FileWatcher: retry helper for inotify timing, accept `:created`/`:changed`
   - WorkflowRunner: `tool_registry_test` changed to `async: false` (was mutating global ETS)
   - All handlers accept `:crashed`/`:timed_out` since Task.yield timing is non-deterministic

## Challenges

1. **Global Mox stubs race with async test modules.** The workflow runner interactive tests failed ~5% of the time because `tool_registry_test` (async: true) deregistered `signal_ready` from the global ETS registry during workflow execution. Fix: `async: false`. Root cause: any test that mutates global state (ETS, Application env, Mox global) must not be async.

2. **`main.ex` is getting large.** At ~1800 lines, the CLI entry point handles 20+ commands with formatters, helpers, and validation logic. The DI pattern keeps it testable but the file is becoming hard to navigate. Consider extracting command modules in a future refactor.

3. **Interactive mode's `signal_ready` + tool-call loop.** When the LLM mock returned `signal_ready` as a tool call, the agent dispatched it, then called the LLM again (tool calls don't terminate the agent loop), creating a 100-iteration loop. Fix: WorkflowRunner stops the agent after `signal_ready`. This was a genuine design bug, not a test issue.

4. **Pre-commit hook failures are disruptive.** When a test flakes during `git commit`, the entire commit fails and must be retried. The hook runs all 6 checks sequentially (~30s total). Consider: parallel check execution, or a `--quick` mode that skips dialyzer for WIP commits.

## Key Insights

1. **"The harness is the product, the tools are plugins."** This emerged from the party mode discussion about missing capabilities (web fetch, LSP). The agent tool registry makes adding new tools trivial — each is one function + one `ToolRegistry.register` call. The architecture decision to separate harness from tools was validated.

2. **Static analysis tools are force multipliers for AI-generated code.** When an AI writes code, it can't run the code to verify it. Static analysis (Dialyzer, Credo, compile warnings) catches errors that would otherwise reach runtime. The pre-commit hook makes this automatic.

3. **Flaky tests must be fixed immediately.** We spent significant time during the sprint debugging and fixing flaky tests. The investment was worth it: 15/15 consecutive passes after fixes. The lesson: never accept "it passes most of the time."

4. **The user-manager → project-manager → specialist agent hierarchy is the right abstraction.** `fam chat` talks to one agent that orchestrates everything else. This matches how humans work with AI assistants — one conversation, delegated execution.

## Patterns Established

1. **`on_handler_error` DI callback** — inject error-reporting function into GenServers that spawn async work, replacing `capture_log` in tests
2. **`write_and_await_event` retry helper** — for inotify timing; write file, wait for event, retry once if missed
3. **`async: false` for global-state-mutating tests** — any test that touches ETS, Application env, or Mox global
4. **Accept `:crashed`/`:timed_out` for Task.yield** — non-deterministic timing means either outcome is valid
5. **Pre-commit hook with 6 checks** — compile, format, credo, sobelow, test, dialyzer
6. **Deadline-based `do_await`** — absolute deadline instead of per-iteration timeout for recursive receive loops
7. **CLI command DI pattern** — `Map.get(deps, :fn_name, &Module.function/arity)` for every command
8. **Stop agent after `signal_ready`** — prevent tool-call loops when WorkflowRunner marks step complete
