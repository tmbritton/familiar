## Successes

- **Architecture held up end-to-end.** Hexagonal ports established in 1.1b were used consistently through 1.7. Mocking externals while using real SQLite felt natural by the integration test.
- **Code review cycle caught real issues.** 48 patches across 9 stories — atom exhaustion (1.6), compensating deletes (1.2), stream lifecycle bugs (1.2, 1.3a), error classification fixes (1.2), and dimension validation (1.2).
- **Test progression was steady.** 8 → 27 → 62 → 117 → 176 → 260 → 291 → 327 → 338. Never a regression.
- **BMad workflow worked well.** create-story → dev-story → code-review cycle caught issues early and kept momentum.
- **Foundation integration test (1.7) proved the full pipeline.** Scan → classify → extract → embed → store → retrieve with real SQLite + sqlite-vec.

## Challenges

- **Streaming and concurrency were consistently tricky.** NDJSON buffer splitting (1.2), spawn vs spawn_link (1.2), Task.async_stream timeout handling (1.4). Recurring theme across 3 stories.
- **Signal trapping deferred.** SIGINT/Ctrl+C cleanup (FR7b) deferred in both 1.4 and 1.7. `run_with_cleanup` handles exceptions but not OS signals. Known gap.
- **Interactive convention review deferred.** CLI `fam conventions review` lists only — no accept/edit/reject flow. Deferred to Epic 7 web UI.

## Key Insights

1. **Compensating transactions solve atomicity at MVP scale.** Simple `Repo.delete` on failure is cleaner than full transaction rollback for multi-step operations (insert → embed → store vector).
2. **Function injection (deps map) works well for orchestration testing.** Cleaner than multiple behaviour ports for CLI/daemon coordination logic.
3. **sqlite-vec requires JSON array strings, not binary encoding.** Critical spike finding from 1.1a that informed all subsequent vector work.
4. **Structural convention analysis provides a useful LLM-free baseline.** Phase 1 (structural) + Phase 2 (LLM) two-phase approach means conventions still discovered when LLM is unavailable.
