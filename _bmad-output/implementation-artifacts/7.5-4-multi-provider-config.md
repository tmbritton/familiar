# Story 7.5-4: Multi-Provider Config and Improved Init

Status: ready-for-dev

## Story

As a developer setting up Familiar,
I want `fam init` to create a config with multiple LLM provider sections and `fam chat --provider <name>` to switch between them,
so that I can configure DeepSeek, Qwen, Ollama, etc. in one file and switch easily.

## Acceptance Criteria

### AC1: Config.toml Supports Multiple Providers

**Given** a generated config.toml
**When** multiple `[providers.xxx]` sections exist with `default = true` on one
**Then** the default provider is used for all commands
**And** `--provider <name>` overrides the default per-command

### AC2: Provider Resolution from Config.toml

**Given** a config.toml with provider settings
**When** the app starts
**Then** the adapter reads API key, base URL, and model from the config file
**And** env vars (`FAMILIAR_API_KEY` etc.) override config.toml values

### AC3: `fam init` Creates Full Project Structure

**Given** a project directory without `.familiar/`
**When** the user runs `fam init`
**Then** `.familiar/` is created with roles, skills, workflows, and config.toml
**And** config.toml has provider sections ready to fill in
**And** project language is auto-detected and populated

### AC4: `fam chat --provider <name>` Overrides Default

**Given** multiple providers configured
**When** the user runs `fam chat --provider qwen`
**Then** the Qwen provider settings are used for that session

### AC5: Test Baseline

**Given** all changes
**When** `mix test` runs
**Then** 0 failures, Credo: 0, Dialyzer: 0
